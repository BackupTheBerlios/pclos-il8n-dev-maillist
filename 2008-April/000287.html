I think the desktop-files tree is a real mess. Translating .desktop files this 
way is a suicide. This task must be done by strictly dividing original desktop 
files and localizations. I think intltool is a proper tool for doing this.

I sat down and wrote some scripts that make the transition easy.

Script make_intltool_ready first corrects lots of problems found in current 
.desktop files, then calls intltool-prepare to generate the infrastructure.
Then intltool-update is used to generate the desktop-files.pot file and finally 
all locales are processed to contain all available msgids.
At the end, there should be NO files with .desktop extension, only files with 
.desktop.in extension. These files do not contain any translations, all the 
translations are in po/$LOCALE.po files.

If you have a new .desktop file and want to add it, just place it on proper 
location and call the second script - make_intltool_update. That script should 
incorporate all messages from that new .desktop file into existing .po files, 
updates desktop-files.pot and converts that new .desktop file into new 
.desktop.in file.

And last but not least, if you want to build the real .desktop files with 
current translations, just call the third script - build_desktop_files.
This script should generate complete desktop-file tree into directory 
../desktop-files-build. This tree contains only the real .desktop files with 
translations.


Please note that you have to install intltool package before launching the scripts.
