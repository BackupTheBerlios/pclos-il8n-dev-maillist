the framework used to translate string using po files in sftwares like 
vit only translate what is translated in the po file ... in your 
personal situation, the po file containing the 8x string was considered 
as lacking the 7x string to the framework (gettext) let the original 
english version of the string in place. the same would have happened if 
you had 1.2 version of vit instaled wile using the 1.1 version of the po 
... you would have been able to read the strings in english but not in 
sv when looking at 8X tab ...

this is the strenght and weakness of this system ... al translations are 
not necessary, but a tiny change of 1 digit in the original code can 
result in untranslated strings.
