a note for the translators ...

i'm happy to see that the flow of updates came back to its previous 
state, nevertheless, i think we should focuss a bit on our main goal. 
This goal is to translate pclinuxos, and not every single software that 
the linux world offers ... thus, i think that we should work on the 
translation yet open and started, finish them and then go back to 
fetching new ones ... i also think that we should focuss on the core 
files and softwares that makes pclos as it is and makes it gr8. so 
focussing on the linux kernel output messages during compilation are not 
must have for the average joe and thus are not necessary to be included 
in the cvs ... no ? i'm looking forward to read some feedback and 
comments on that point ...

anyway, in plain text, the holly bible is 1 mb big ... so i find it 
quite hillarious that the linux kernel po is 2 mb big :)

last but not least, remove version and country tags from your po 
submission file name. We only need the latest version and if you mention 
the locale in the mail , i'll obviously figure out the actual locale i 
must attach it to :) sometimes the version numbers/locale tag in 
filenames can be puzzeling and this lead me to test hell when i validate 
the file you submitted to me !

thanks for your attention
